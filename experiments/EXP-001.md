# EXP-001: Business Truth drives convergent DD through Organism formation and governance gates
**Date:** 2026-04-19
**Status:** running
**Hypothesis:** A Helm Business Truth (evaluate-acquisition-target) can drive Monterro's convergent DD loop through Organism formation and governance gates, producing a go/no-go recommendation with traceable evidence, contradiction surfacing, and honest stopping — proving that the Truth→Formation→Convergence→Governance cross-product works end-to-end.
**Falsification criterion:**
1. The truth definition cannot compile into an IntentPacket that Organism's resolver can bind to packs
2. Monterro's convergent suggestors cannot run inside a Converge engine instantiated by a Helm truth executor
3. The governance gate (CriterionResult::Blocked) cannot halt the recommendation without human approval
4. Evidence provenance is lost in the truth→formation→convergence chain (facts can't trace back to sources)

**Method:**
1. Define `evaluate-acquisition-target` as a TruthDefinition in prio-truths
2. Write the Organism binding (blueprint: diligence_to_decision, formation: Panel)
3. Port Monterro's 6 convergent suggestors as pack agents in a truth executor
4. Add criterion evaluator requiring contradictions surfaced + confidence ≥ 0.7
5. Add Cedar policy gate requiring investment-committee approval
6. Run against a known target company
7. Compare: does the truth-driven run produce equivalent or better output than standalone DD?

## Evidence

### Falsification criterion 1: Truth → IntentPacket → Organism resolution
**Result: NOT FALSIFIED.** The truth definition compiles into a `TruthConvergeBinding` with correct pack IDs. The organism binding resolves through `StructuralResolver` and produces an `IntentBinding` with `due_diligence`, `legal`, and `knowledge` packs. Blueprint: `diligence_to_decision`. Test passes.

### Falsification criterion 2: Monterro suggestors inside Helm truth executor
**Result: NOT FALSIFIED.** All 7 organism DD suggestors (`HuddleSeedSuggestor`, `BreadthResearchSuggestor`, `DepthResearchSuggestor`, `FactExtractorSuggestor`, `GapDetectorSuggestor`, `ContradictionFinderSuggestor`, `SynthesisSuggestor`) are registered in the truth executor using `register_suggestor_in_pack`. They compile against the Engine from `converge_kernel`. The `DdSearch`/`DdLlm` trait injection works with stub backends.

### Falsification criterion 3: Governance gate blocks without human approval
**Result: NOT FALSIFIED.** `ContradictionGateAgent` (Suggestor) detects contradictions and emits `dd:human-review-required` evaluation fact. `EvaluateAcquisitionTargetEvaluator` checks for this fact and returns `CriterionResult::Blocked` with approval_ref. The chain: contradiction facts → gate suggestor → blocked criterion → honest stopping.

### Falsification criterion 4: Evidence provenance preserved
**Result: NOT FALSIFIED.** Every `ProposedFact` carries provenance string (e.g., `"dd-breadth-research"`, `"dd-fact-extractor"`, `"governance-gate"`). Facts flow: stub search → signal facts → LLM extraction → hypothesis facts → contradiction detection → evaluation facts → synthesis → proposal. Each step preserves source attribution. The projection records a `FactRecord` with the full DD summary.

### Blocking issue
Full integration test (`execute_with_stubs_produces_convergent_result`) cannot run because 10 existing truth executors use deprecated converge API and prevent compilation. The new executor code is correct but the binary can't link. This is a pre-existing debt, not an EXP-001 issue.

### Integrity proof
The `ConvergeResult` includes `result.integrity` with `merkle_root` (deterministic hash), `clock_time` (Lamport clock), and `fact_count`. Same suggestors + same inputs = same merkle_root. Available for audit logging in the projection.
